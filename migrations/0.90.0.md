## Mastering Foundation — 24/96 Streaming-Grade Delivery

This release lands the foundation phase for automated album-coherence mastering (issue [#290](https://github.com/bitwize-music-studio/claude-ai-music-skills/issues/290)). Most users will see the format change on the next `master_album` run.

### What changed for me as a user?

**If you never touched mastering config**: your next mastered output will be 24-bit at 96 kHz. Files are ~3x larger but eligible for Apple Hi-Res Lossless and Tidal Max badges. Spotify/YouTube still downsample to 44.1 kHz.

**If you had custom `{overrides}/mastering-presets.yaml`**: your per-genre overrides continue to win. If you explicitly set `output_bits: 16` in an override, that still produces 16-bit output for that genre.

**If you want to stay at 16/44.1**: add the config snippet from the manual-action box above.

### Why 24/96 specifically?

- **Apple Hi-Res Lossless badge**: requires ≥24-bit AND >48 kHz (strict >). 48 kHz doesn't qualify; 96 kHz does.
- **Tidal Max badge**: requires 24-bit AND >44.1 kHz (strict >).
- **Spotify/YouTube**: don't care about resolution, normalize at 44.1 kHz regardless.
- **DistroKid single-ingest** accepts up to 192 kHz; 96 is the sweet spot.

The honesty caveat: Suno outputs at 44.1 kHz. Upsampling to 96 kHz doesn't add audio information — it fills the file with silence above ~22 kHz. Every `master_album` run now emits a notice in the response when the delivery rate exceeds the source rate, so this is visible at runtime too.

### New opt-in archival

Setting `mastering.archival_enabled: true` adds a stage that writes 32-bit float copies of each mastered track to `archival/`. Intended for re-mastering spec changes later (e.g., re-release with different targets) without re-polishing stems. Adds ~1-2 GB per 12-track album when on. Clean up with `prune_archival` (keeps the N most-recent files by mtime).

### Subsequent phases

This is phase 1a of a 10-phase rollout. Next phases land:
- **1b**: multi-metric `analyze_audio` (STL-95, low-RMS, vocal-RMS)
- **2a–c**: anchor selector, coherence correction, signature persistence
- **3a–c**: LAYOUT.md generator, ADM validation + ceiling guard, metadata MVP
- **4a–b**: `measure_album_signature` tool, integration + docs

Companion issue [#303](https://github.com/bitwize-music-studio/claude-ai-music-skills/issues/303) tracks the full-fidelity metadata embedding work (verification round-trip, post-release ISRC/UPC capture, extended credits, character encoding).
